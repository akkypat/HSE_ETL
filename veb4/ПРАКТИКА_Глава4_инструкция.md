Открой PowerShell и подключись к psql:

### 1.1 Общая статистика данных

```sql
SELECT
    COUNT(*)                                         AS total_rows,
    COUNT(DISTINCT station_id)                       AS cities,
    MIN(observed_at)::DATE                           AS date_from,
    MAX(observed_at)::DATE                           AS date_to,
    COUNT(*) FILTER (WHERE qc_flag = 0)              AS rows_ok,
    COUNT(*) FILTER (WHERE qc_flag = 1)              AS rows_out_of_range,
    COUNT(*) FILTER (WHERE qc_flag = 2)              AS rows_missing,
    ROUND(100.0 * COUNT(*) FILTER (WHERE qc_flag=0)
          / NULLIF(COUNT(*),0), 2)                   AS pct_ok
FROM staging_observations;
```

> 📸 **Скриншот для Рис. 4.1** — сделай скриншот результата этого запроса.
> Подпись: «Рис. 4.1 — Статистика данных в таблице staging_observations»

### 1.2 Число записей по городам

```sql
SELECT
    d.city_name,
    COUNT(*)                            AS obs_count,
    MIN(o.observed_at)::DATE            AS date_from,
    MAX(o.observed_at)::DATE            AS date_to
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
GROUP BY d.city_name
ORDER BY d.city_name;
```

Запиши значения `obs_count` для каждого города — они войдут в таблицу 4.3 диплома.

### 1.3 Размер хранилища

```sql
SELECT
    relname                                         AS table_name,
    pg_size_pretty(pg_total_relation_size(oid))     AS total_size
FROM pg_class
WHERE relname IN (
    'staging_observations','mart_hourly_weather',
    'mart_daily_summary','mart_forecast','data_quality_log'
)
ORDER BY pg_total_relation_size(oid) DESC;
```

### 1.4 Число строк в агрегатах

```sql
SELECT 'mart_hourly_weather' AS view_name, COUNT(*) AS rows,
       MIN(bucket)::DATE AS from_dt, MAX(bucket)::DATE AS to_dt
FROM mart_hourly_weather
UNION ALL
SELECT 'mart_daily_summary', COUNT(*), MIN(bucket)::DATE, MAX(bucket)::DATE
FROM mart_daily_summary
UNION ALL
SELECT 'mart_forecast', COUNT(*), MIN(target_at)::DATE, MAX(target_at)::DATE
FROM mart_forecast;
```

## Блок 2 — Измерение производительности (Таблица 4.3)

### 2.1 Время запроса витрины за 7 дней

```sql
\timing on
SELECT station_id, bucket, avg_temperature_c
FROM mart_hourly_weather
WHERE bucket > NOW() - INTERVAL '7 days'
ORDER BY station_id, bucket;
\timing off
```

Запиши время выполнения (строка `Time: xxx.xxx ms`).

### 2.2 Время запроса витрины за 30 дней

```sql
\timing on
SELECT station_id, bucket, avg_temperature_c, avg_pressure_hpa
FROM mart_hourly_weather
WHERE bucket > NOW() - INTERVAL '30 days'
ORDER BY station_id, bucket;
\timing off
```

### 2.3 Аналитический запрос — амплитуда температур

```sql
\timing on
SELECT
    d.city_name,
    ROUND(MIN(s.min_temp_c)::NUMERIC,1)                       AS abs_min,
    ROUND(AVG(s.avg_temp_c)::NUMERIC,1)                       AS avg_temp,
    ROUND(MAX(s.max_temp_c)::NUMERIC,1)                       AS abs_max,
    ROUND((MAX(s.max_temp_c)-MIN(s.min_temp_c))::NUMERIC,1)  AS amplitude_c,
    ROUND(SUM(s.total_precip_mm)::NUMERIC,1)                 AS total_precip_mm
FROM mart_daily_summary s
JOIN dim_station d ON s.station_id = d.station_id
WHERE s.bucket >= NOW() - INTERVAL '30 days'
GROUP BY d.city_name
ORDER BY amplitude_c DESC;
\timing off
```

## Блок 3 — Метрики data_quality_check (Таблица 4.4)

### 3.1 Последние результаты проверок

```sql
SELECT
    check_id,
    status,
    detail,
    checked_at AT TIME ZONE 'Europe/Moscow' AS checked_msk
FROM data_quality_log
WHERE checked_at = (SELECT MAX(checked_at) FROM data_quality_log)
ORDER BY check_id;
```

> 📸 **Скриншот для Рис. 3.6** — таблица с 5 строками PASS.

### 3.2 Статистика проверок за последние 24 часа

```sql
SELECT
    check_id,
    COUNT(*) FILTER (WHERE status='PASS') AS pass_count,
    COUNT(*) FILTER (WHERE status='WARN') AS warn_count,
    COUNT(*) FILTER (WHERE status='FAIL') AS fail_count
FROM data_quality_log
WHERE checked_at > NOW() - INTERVAL '24 hours'
GROUP BY check_id
ORDER BY check_id;
```

## Блок 4 — Метрики ML-модели (Таблица 4.7)

```sql
SELECT
    city_name                       AS "Город",
    ROUND(rmse::NUMERIC,2)          AS "RMSE °C",
    ROUND(mae::NUMERIC,2)           AS "MAE °C",
    ROUND(r2::NUMERIC,3)            AS "R²",
    train_rows                      AS "Строк обучения",
    trained_at::DATE                AS "Дата обучения"
FROM ml_model_metrics
WHERE trained_at = (SELECT MAX(trained_at) FROM ml_model_metrics)
ORDER BY rmse ASC;
```

Эти значения вставь в таблицу 4.7 диплома.

### 4.1 Прогноз для Москвы (пример)

```sql
SELECT
    target_at AT TIME ZONE 'Europe/Moscow'  AS "Время (МСК)",
    ROUND(temperature_c::NUMERIC,1)         AS "Прогноз °C"
FROM mart_forecast f
WHERE station_id = 1
  AND forecast_at = (
      SELECT MAX(forecast_at) FROM mart_forecast WHERE station_id=1
  )
ORDER BY target_at;
```

## Блок 5 — Верификация REST API (Таблица 4.4, EXT-07–11)

Выйди из psql (`\q`) и выполни в PowerShell:

```powershell
# EXT-07: /health
curl http://localhost:8000/health

# EXT-08: /current — все 8 городов
curl http://localhost:8000/current

# EXT-09: /history — история за 7 дней
curl "http://localhost:8000/history/%D0%9C%D0%BE%D1%81%D0%BA%D0%B2%D0%B0?days=7"

# EXT-10: /compare — сравнение городов
curl "http://localhost:8000/compare?days=30"

# EXT-11: /anomalies — аномалии
curl "http://localhost:8000/anomalies?sigma=3"
```

Для каждого запроса зафиксируй:
- HTTP-статус (должен быть 200)
- Время ответа (curl показывает `time_total`)

Для красивого вывода добавь `| python -m json.tool`:
```powershell
curl http://localhost:8000/health | python -m json.tool
```
