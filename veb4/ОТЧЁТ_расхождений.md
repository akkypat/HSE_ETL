## РАСХОЖДЕНИЕ №1 🔴 КРИТИЧНО

### Время выполнения IT-07 — внутреннее противоречие в тексте

**Где:** Таблица 4.2 (строка IT-07) vs абзац после неё

**Проблема:** В таблице 4.2 указано **8 мин**, в абзаце ниже написано **18 минут**.

> Таблица 4.2, строка IT-07:  
> `IT-07 | Историческая загрузка backfill | 2025-01-01 — вчера | ~120 000 записей, 0 ошибок | 8 мин`

> Абзац после таблицы 4.2:  
> «загрузка данных за 16 месяцев... заняла **18 минут** и сформировала ~120 000 записей»

**Что исправить:** Оба значения должны совпадать. По логике ~120 000 строк за 16 месяцев реалистичнее занимают 15–20 минут, поэтому **18 мин** в тексте достовернее.

```
Таблица 4.2, столбец «Время», строка IT-07:
БЫЛО:   8 мин
СТАЛО:  18 мин
```

## РАСХОЖДЕНИЕ №10 🟡 ВАЖНО

### Путь Raw-слоя в тексте не соответствует коду

**Где:** Глава 2, раздел 2.2.2 (описание Raw-слоя)

**Проблема:** В тексте документа описан путь `/data/raw/{источник}/{город}/{YYYY-MM-DD}/{HH}.json`. В коде (`hourly_weather_etl.py`) реальный путь — `/opt/airflow/raw_data/open-meteo/{city_id}/{YYYY-MM-DD}/{HH}.json.gz`.

Три отличия:
1. Корневая директория: `/data/raw/` → `/opt/airflow/raw_data/`
2. Идентификатор города: `{город}` (название) → `{city_id}` (числовой ID 1–8)
3. Формат файла: `.json` → `.json.gz` (gzip-сжатие)

**Что исправить:**
```
БЫЛО:   «/data/raw/{источник}/{город}/{YYYY-MM-DD}/{HH}.json»
СТАЛО:  «/opt/airflow/raw_data/open-meteo/{city_id}/{YYYY-MM-DD}/{HH}.json.gz»

Пример: /opt/airflow/raw_data/open-meteo/1/2025-05-11/14.json.gz
```
