## 2. Установка необходимого ПО

### 2.1 Включение WSL2

Открой **PowerShell от имени администратора** (ПКМ → «Запуск от имени администратора») и выполни:

```powershell
wsl --install
```

После установки **перезагрузи компьютер**. WSL2 установится автоматически с Ubuntu 22.04.

Проверь версию:
```powershell
wsl --version
```

Ожидаемый вывод:
```
WSL version: 2.x.x.x
Kernel version: 5.15.x.x
```

### 2.2 Docker Desktop

1. Скачай установщик: https://www.docker.com/products/docker-desktop/  
   Файл: `Docker Desktop Installer.exe` (≈600 МБ)

2. Запусти установщик. При установке убедись, что отмечен чекбокс:  
   ✅ **Use WSL 2 instead of Hyper-V (recommended)**

3. После установки **перезагрузи компьютер**.

4. Запусти Docker Desktop. Дождись статуса **«Docker Desktop is running»** (зелёная иконка в трее).

5. Проверь установку в PowerShell:
```powershell
docker --version
docker compose version
```

Ожидаемый вывод:
```
Docker version 26.1.4, build 5650f9b
Docker Compose version v2.27.1
```

> ⚠️ Нужен именно **Docker Compose v2** (команда `docker compose`, без дефиса). Если у тебя старая версия с `docker-compose` — обнови Docker Desktop.

### 2.3 Python 3.11

1. Скачай: https://www.python.org/downloads/release/python-3119/  
   Файл: `python-3.11.9-amd64.exe`

2. Запусти установщик.  
   ✅ Обязательно отметь **«Add Python 3.11 to PATH»** в самом низу первого окна.

3. Выбери **«Customize installation»** → нажми Next → отметь ✅ **«Install for all users»** → Install.

4. Проверь:
```powershell
python --version
pip --version
```

Ожидаемый вывод:
```
Python 3.11.9
pip 24.x.x from ...
```

### 2.4 Visual Studio Code (редактор кода)

1. Скачай: https://code.visualstudio.com/  
   Файл: `VSCodeUserSetup-x64-x.xx.x.exe`

2. Установи с настройками по умолчанию.

3. Установи расширение **Python** (через меню Extensions → поиск «Python» → Install).

### 2.5 DBeaver (просмотр базы данных, опционально)

Для удобного просмотра таблиц PostgreSQL можно использовать DBeaver:  
Скачай: https://dbeaver.io/download/ → Community Edition → Windows installer.

### 4.3 `init.sql` — схема базы данных

Файл: `C:\weather_pipeline\init.sql`

```sql
-- ── Включение расширения TimescaleDB ──────────────────────────
CREATE EXTENSION IF NOT EXISTS timescaledb;

-- ── Справочник городов ────────────────────────────────────────
CREATE TABLE IF NOT EXISTS dim_station (
    station_id   SERIAL PRIMARY KEY,
    city_name    VARCHAR(100) NOT NULL,
    city_name_en VARCHAR(100) NOT NULL,
    latitude     DOUBLE PRECISION NOT NULL CHECK (latitude  BETWEEN -90  AND 90),
    longitude    DOUBLE PRECISION NOT NULL CHECK (longitude BETWEEN -180 AND 180),
    elevation_m  REAL,
    timezone     VARCHAR(50) NOT NULL,
    country      VARCHAR(50) NOT NULL DEFAULT 'Russia'
);

-- ── Наполнение справочника ────────────────────────────────────
INSERT INTO dim_station (city_name, city_name_en, latitude, longitude, elevation_m, timezone) VALUES
    ('Москва',          'Moscow',          55.7558,  37.6173,  156.0, 'Europe/Moscow'),
    ('Санкт-Петербург', 'Saint Petersburg',59.9343,  30.3351,    8.0, 'Europe/Moscow'),
    ('Екатеринбург',    'Yekaterinburg',   56.8519,  60.6122,  269.0, 'Asia/Yekaterinburg'),
    ('Новосибирск',     'Novosibirsk',     54.9924,  82.9041,  162.0, 'Asia/Novosibirsk'),
    ('Казань',          'Kazan',           55.8304,  49.0661,   74.0, 'Europe/Moscow'),
    ('Краснодар',       'Krasnodar',       45.0355,  38.9753,   34.0, 'Europe/Moscow'),
    ('Красноярск',      'Krasnoyarsk',     56.0153,  92.8932,  277.0, 'Asia/Krasnoyarsk'),
    ('Владивосток',     'Vladivostok',     43.1155, 131.8855,   25.0, 'Asia/Vladivostok');

-- ── Справочник дат ────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS dim_date (
    date_id     DATE PRIMARY KEY,
    year        SMALLINT NOT NULL,
    month       SMALLINT NOT NULL,
    day         SMALLINT NOT NULL,
    day_of_week SMALLINT NOT NULL,  -- 0=Mon ... 6=Sun
    quarter     SMALLINT NOT NULL,
    season      VARCHAR(10) NOT NULL
);

-- Заполнение dim_date с 2022-01-01 по 2027-12-31
INSERT INTO dim_date (date_id, year, month, day, day_of_week, quarter, season)
SELECT
    d::DATE,
    EXTRACT(YEAR  FROM d)::SMALLINT,
    EXTRACT(MONTH FROM d)::SMALLINT,
    EXTRACT(DAY   FROM d)::SMALLINT,
    EXTRACT(ISODOW FROM d)::SMALLINT - 1,
    EXTRACT(QUARTER FROM d)::SMALLINT,
    CASE
        WHEN EXTRACT(MONTH FROM d) IN (12, 1, 2)  THEN 'Зима'
        WHEN EXTRACT(MONTH FROM d) IN (3, 4, 5)   THEN 'Весна'
        WHEN EXTRACT(MONTH FROM d) IN (6, 7, 8)   THEN 'Лето'
        ELSE 'Осень'
    END
FROM generate_series('2022-01-01'::DATE, '2027-12-31'::DATE, '1 day'::INTERVAL) AS d
ON CONFLICT (date_id) DO NOTHING;

-- ── Основная таблица наблюдений (гипертаблица TimescaleDB) ────
CREATE TABLE IF NOT EXISTS staging_observations (
    obs_id              BIGSERIAL,
    station_id          INTEGER NOT NULL REFERENCES dim_station(station_id),
    observed_at         TIMESTAMPTZ NOT NULL,
    temperature_c       REAL,        -- температура воздуха, °C
    apparent_temp_c     REAL,        -- ощущаемая температура, °C
    pressure_hpa        REAL,        -- атм. давление на уровне моря, гПа
    humidity_pct        REAL,        -- относительная влажность, %
    wind_speed_ms       REAL,        -- скорость ветра на высоте 10 м, м/с
    wind_direction_deg  SMALLINT,    -- направление ветра, 0–360°
    precipitation_mm    REAL,        -- осадки за час, мм
    data_source         VARCHAR(32)  NOT NULL DEFAULT 'open-meteo',
    qc_flag             SMALLINT     NOT NULL DEFAULT 0,  -- 0=OK, 1=out_of_range, 2=missing
    PRIMARY KEY (obs_id, observed_at)
);

-- Преобразование в гипертаблицу (партиции по 1 неделе)
SELECT create_hypertable(
    'staging_observations',
    'observed_at',
    chunk_time_interval => INTERVAL '1 week',
    if_not_exists => TRUE
);

-- Составной индекс для типичных запросов (город + период)
CREATE INDEX IF NOT EXISTS idx_obs_station_time
    ON staging_observations (station_id, observed_at DESC);

-- ── Витрина: почасовые агрегаты ───────────────────────────────
CREATE MATERIALIZED VIEW mart_hourly_weather
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 hour', observed_at)  AS bucket,
    station_id,
    AVG(temperature_c)::REAL            AS avg_temperature_c,
    MIN(temperature_c)::REAL            AS min_temperature_c,
    MAX(temperature_c)::REAL            AS max_temperature_c,
    AVG(apparent_temp_c)::REAL          AS avg_apparent_temp_c,
    AVG(pressure_hpa)::REAL             AS avg_pressure_hpa,
    AVG(humidity_pct)::REAL             AS avg_humidity_pct,
    SUM(precipitation_mm)::REAL         AS total_precipitation_mm,
    AVG(wind_speed_ms)::REAL            AS avg_wind_speed_ms,
    COUNT(*)                            AS obs_count,
    COUNT(*) FILTER (WHERE qc_flag > 0) AS bad_obs_count
FROM staging_observations
WHERE qc_flag < 2
GROUP BY bucket, station_id
WITH NO DATA;

-- Политика автообновления почасового агрегата
SELECT add_continuous_aggregate_policy(
    'mart_hourly_weather',
    start_offset    => INTERVAL '3 hours',
    end_offset      => INTERVAL '1 hour',
    schedule_interval => INTERVAL '1 hour',
    if_not_exists   => TRUE
);

-- ── Витрина: суточные агрегаты ────────────────────────────────
CREATE MATERIALIZED VIEW mart_daily_summary
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 day', observed_at)   AS bucket,
    station_id,
    AVG(temperature_c)::REAL            AS avg_temp_c,
    MIN(temperature_c)::REAL            AS min_temp_c,
    MAX(temperature_c)::REAL            AS max_temp_c,
    AVG(pressure_hpa)::REAL             AS avg_pressure_hpa,
    AVG(humidity_pct)::REAL             AS avg_humidity_pct,
    SUM(precipitation_mm)::REAL         AS total_precip_mm,
    AVG(wind_speed_ms)::REAL            AS avg_wind_speed_ms
FROM staging_observations
WHERE qc_flag < 2
GROUP BY bucket, station_id
WITH NO DATA;

-- Политика автообновления суточного агрегата
SELECT add_continuous_aggregate_policy(
    'mart_daily_summary',
    start_offset    => INTERVAL '2 days',
    end_offset      => INTERVAL '1 hour',
    schedule_interval => INTERVAL '1 day',
    if_not_exists   => TRUE
);

-- ── Политика сжатия старых данных ─────────────────────────────
ALTER TABLE staging_observations SET (
    timescaledb.compress,
    timescaledb.compress_segmentby = 'station_id'
);

SELECT add_compression_policy(
    'staging_observations',
    INTERVAL '7 days',
    if_not_exists => TRUE
);
```

"""
DAG: hourly_weather_etl
========================
Почасовой сбор метеорологических данных из Open-Meteo API
для 8 крупных городов России.

Граф задач для каждого города:
  check_api → [extract_<city> → transform_<city>] → aggregate_hourly

Расписание: каждый час (0 * * * *)
Автор: weather_team
"""

from __future__ import annotations

import gzip
import json
import logging
import os
from datetime import datetime, timedelta, timezone
from pathlib import Path

import psycopg2
import psycopg2.extras
import requests
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.providers.postgres.operators.postgres import PostgresOperator
from airflow.providers.http.sensors.http import HttpSensor
from airflow.utils.task_group import TaskGroup

# ── Конфигурация городов ──────────────────────────────────────────────────────
CITIES = [
    {"id": 1, "name": "Москва",          "lat": 55.7558,  "lon": 37.6173,  "tz": "Europe/Moscow"},
    {"id": 2, "name": "Санкт-Петербург", "lat": 59.9343,  "lon": 30.3351,  "tz": "Europe/Moscow"},
    {"id": 3, "name": "Екатеринбург",    "lat": 56.8519,  "lon": 60.6122,  "tz": "Asia/Yekaterinburg"},
    {"id": 4, "name": "Новосибирск",     "lat": 54.9924,  "lon": 82.9041,  "tz": "Asia/Novosibirsk"},
    {"id": 5, "name": "Казань",          "lat": 55.8304,  "lon": 49.0661,  "tz": "Europe/Moscow"},
    {"id": 6, "name": "Краснодар",       "lat": 45.0355,  "lon": 38.9753,  "tz": "Europe/Moscow"},
    {"id": 7, "name": "Красноярск",      "lat": 56.0153,  "lon": 92.8932,  "tz": "Asia/Krasnoyarsk"},
    {"id": 8, "name": "Владивосток",     "lat": 43.1155,  "lon": 131.8855, "tz": "Asia/Vladivostok"},
]

# ── API-параметры Open-Meteo ──────────────────────────────────────────────────
OPEN_METEO_URL = "https://api.open-meteo.com/v1/forecast"
HOURLY_VARS = [
    "temperature_2m",
    "apparent_temperature",
    "pressure_msl",
    "relative_humidity_2m",
    "wind_speed_10m",
    "wind_direction_10m",
    "precipitation",
]

# ── Физические диапазоны для QC-проверок (по ВМО CIMO Guide) ─────────────────
QC_RANGES = {
    "temperature_c":      (-80.0,  60.0),
    "apparent_temp_c":    (-90.0,  70.0),
    "pressure_hpa":       (870.0, 1084.0),
    "humidity_pct":       (  0.0,  100.0),
    "wind_speed_ms":      (  0.0,   90.0),
    "wind_direction_deg": (  0,     360),
    "precipitation_mm":   (  0.0,  300.0),
}

# ── Параметры подключения к БД (из переменных окружения контейнера) ───────────
def _get_db_conn():
    return psycopg2.connect(
        host=os.environ["WEATHER_DB_HOST"],
        port=int(os.environ.get("WEATHER_DB_PORT", 5432)),
        dbname=os.environ["WEATHER_DB_NAME"],
        user=os.environ["WEATHER_DB_USER"],
        password=os.environ["WEATHER_DB_PASSWORD"],
        connect_timeout=10,
    )

# ── QC-функция ────────────────────────────────────────────────────────────────
def _apply_qc(field: str, value) -> tuple:
    """
    Проверяет значение на допустимый физический диапазон.
    Возвращает (очищенное_значение, qc_flag):
      0 — норма, 1 — вне диапазона, 2 — отсутствует.
    """
    if value is None:
        return None, 2
    lo, hi = QC_RANGES.get(field, (-1e9, 1e9))
    if not (lo <= value <= hi):
        logging.warning("QC fail: %s = %s (допустимо %s–%s)", field, value, lo, hi)
        return None, 1
    return value, 0

# ── ЗАДАЧА 1: Извлечение данных из Open-Meteo API ────────────────────────────
def extract_weather(city: dict, **context) -> str:
    """
    Запрашивает почасовые данные для города из Open-Meteo API.
    Сохраняет сырой JSON в gzip-файл на Raw-слое.
    Возвращает путь к файлу (передаётся в XCom).
    """
    params = {
        "latitude":     city["lat"],
        "longitude":    city["lon"],
        "hourly":       ",".join(HOURLY_VARS),
        "timezone":     city["tz"],
        "forecast_days": 1,
        "models":       "best_match",
    }

    resp = requests.get(OPEN_METEO_URL, params=params, timeout=30)
    resp.raise_for_status()

    now = datetime.now(timezone.utc)
    raw_dir = Path("/opt/airflow/raw_data") / "open-meteo" / str(city["id"]) / now.strftime("%Y-%m-%d")
    raw_dir.mkdir(parents=True, exist_ok=True)
    file_path = raw_dir / f"{now.strftime('%H')}.json.gz"

    with gzip.open(file_path, "wt", encoding="utf-8") as f:
        json.dump(resp.json(), f, ensure_ascii=False)

    logging.info("Сохранён Raw-файл: %s", file_path)
    return str(file_path)

# ── ЗАДАЧА 2: Трансформация и загрузка в Staging ─────────────────────────────
def transform_and_load(city: dict, **context) -> int:
    """
    Читает сырой JSON из Raw-слоя, применяет QC-проверки,
    нормализует данные и записывает в staging_observations.
    Возвращает количество вставленных строк.
    """
    # Читаем путь к файлу из XCom предыдущей задачи
    ti = context["ti"]
    file_path = ti.xcom_pull(task_ids=f"city_{city['id']}.extract")

    with gzip.open(file_path, "rt", encoding="utf-8") as f:
        raw = json.load(f)

    hourly = raw.get("hourly", {})
    times  = hourly.get("time", [])

    if not times:
        logging.warning("Пустой ответ API для города %s", city["name"])
        return 0

    records = []
    for i, ts_str in enumerate(times):
        observed_at = datetime.fromisoformat(ts_str).replace(tzinfo=timezone.utc)

        # Маппинг полей API → поля БД
        raw_vals = {
            "temperature_c":      hourly.get("temperature_2m",      [None] * len(times))[i],
            "apparent_temp_c":    hourly.get("apparent_temperature", [None] * len(times))[i],
            "pressure_hpa":       hourly.get("pressure_msl",         [None] * len(times))[i],
            "humidity_pct":       hourly.get("relative_humidity_2m", [None] * len(times))[i],
            "wind_speed_ms":      hourly.get("wind_speed_10m",        [None] * len(times))[i],
            "wind_direction_deg": hourly.get("wind_direction_10m",   [None] * len(times))[i],
            "precipitation_mm":   hourly.get("precipitation",         [None] * len(times))[i],
        }

        # QC-проверки
        clean = {}
        max_flag = 0
        for field, val in raw_vals.items():
            clean_val, flag = _apply_qc(field, val)
            clean[field] = clean_val
            max_flag = max(max_flag, flag)

        records.append((
            city["id"],
            observed_at,
            clean["temperature_c"],
            clean["apparent_temp_c"],
            clean["pressure_hpa"],
            clean["humidity_pct"],
            clean["wind_speed_ms"],
            clean["wind_direction_deg"],
            clean["precipitation_mm"],
            "open-meteo",
            max_flag,
        ))

    # Запись в PostgreSQL (идемпотентно — дубли игнорируются)
    conn = _get_db_conn()
    try:
        with conn, conn.cursor() as cur:
            psycopg2.extras.execute_values(
                cur,
                """
                INSERT INTO staging_observations
                  (station_id, observed_at, temperature_c, apparent_temp_c,
                   pressure_hpa, humidity_pct, wind_speed_ms, wind_direction_deg,
                   precipitation_mm, data_source, qc_flag)
                VALUES %s
                ON CONFLICT (obs_id, observed_at) DO NOTHING
                """,
                records,
                page_size=500,
            )
        logging.info("Город %s: вставлено %d записей", city["name"], len(records))
    finally:
        conn.close()

    return len(records)

# ── SQL для обновления агрегата ───────────────────────────────────────────────
REFRESH_HOURLY_SQL = "CALL refresh_continuous_aggregate('mart_hourly_weather', NULL, NULL);"

# ── Аргументы по умолчанию для DAG ───────────────────────────────────────────
default_args = {
    "owner":                    "weather_team",
    "retries":                  3,
    "retry_delay":              timedelta(minutes=5),
    "retry_exponential_backoff": True,
    "max_retry_delay":          timedelta(minutes=30),
    "email_on_failure":         False,
    "depends_on_past":          False,
}

# ── Определение DAG ───────────────────────────────────────────────────────────
with DAG(
    dag_id="hourly_weather_etl",
    default_args=default_args,
    description="Почасовой сбор погоды по 8 городам России из Open-Meteo API",
    schedule_interval="0 * * * *",   # каждый час в :00
    start_date=datetime(2024, 1, 1),
    catchup=False,
    max_active_runs=1,
    tags=["weather", "etl", "open-meteo", "production"],
    doc_md=__doc__,
) as dag:

    # ── Проверка доступности API ──────────────────────────────────────────────
    check_api = HttpSensor(
        task_id="check_api_availability",
        http_conn_id="open_meteo_api",
        endpoint="v1/forecast?latitude=55.75&longitude=37.62&hourly=temperature_2m&forecast_days=1",
        method="GET",
        response_check=lambda response: response.status_code == 200,
        poke_interval=30,
        timeout=120,
        mode="reschedule",
    )

    # ── Параллельные TaskGroup для каждого города ─────────────────────────────
    city_groups = []
    for city in CITIES:
        with TaskGroup(group_id=f"city_{city['id']}", tooltip=city["name"]) as tg:

            extract = PythonOperator(
                task_id="extract",
                python_callable=extract_weather,
                op_kwargs={"city": city},
            )

            transform = PythonOperator(
                task_id="transform",
                python_callable=transform_and_load,
                op_kwargs={"city": city},
            )

            extract >> transform

        city_groups.append(tg)

    # ── Обновление почасового агрегата ────────────────────────────────────────
    aggregate = PostgresOperator(
        task_id="aggregate_hourly",
        postgres_conn_id="weather_db",

    )

    # ── Граф зависимостей ─────────────────────────────────────────────────────
    check_api >> city_groups >> aggregate

## 8. Проверка данных в TimescaleDB

### 8.1 Подключение через Docker

```powershell
# Открываем psql в контейнере TimescaleDB
docker exec -it weather_pg_data psql -U weather_user -d weather_db
```

### 8.2 Полезные SQL-запросы

Внутри psql выполни следующие команды (каждую завершай `;`):

**Число записей по городам:**
```sql
SELECT d.city_name, COUNT(*) AS obs_count
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
GROUP BY d.city_name
ORDER BY obs_count DESC;
```

**Последние наблюдения (текущие данные):**
```sql
SELECT d.city_name,
       o.observed_at AT TIME ZONE 'Europe/Moscow' AS local_time,
       o.temperature_c,
       o.humidity_pct,
       o.wind_speed_ms,
       o.qc_flag
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
WHERE o.observed_at > NOW() - INTERVAL '2 hours'
  AND o.qc_flag = 0
ORDER BY d.city_name, o.observed_at DESC;
```

**Суточные экстремумы температуры:**
```sql
SELECT d.city_name,
       s.bucket::DATE AS date,
       ROUND(s.min_temp_c::NUMERIC, 1) AS min_c,
       ROUND(s.avg_temp_c::NUMERIC, 1) AS avg_c,
       ROUND(s.max_temp_c::NUMERIC, 1) AS max_c,
       ROUND(s.total_precip_mm::NUMERIC, 1) AS precip_mm
FROM mart_daily_summary s
JOIN dim_station d ON s.station_id = d.station_id
WHERE s.bucket >= NOW() - INTERVAL '7 days'
ORDER BY s.bucket DESC, d.city_name;
```

**Город с максимальной температурной амплитудой за 30 дней:**
```sql
SELECT d.city_name,
       ROUND(MIN(s.min_temp_c)::NUMERIC, 1) AS abs_min,
       ROUND(MAX(s.max_temp_c)::NUMERIC, 1) AS abs_max,
       ROUND((MAX(s.max_temp_c) - MIN(s.min_temp_c))::NUMERIC, 1) AS amplitude
FROM mart_daily_summary s
JOIN dim_station d ON s.station_id = d.station_id
WHERE s.bucket >= NOW() - INTERVAL '30 days'
GROUP BY d.city_name
ORDER BY amplitude DESC;
```

**Статистика качества данных (QC-флаги):**
```sql
SELECT d.city_name,
       o.qc_flag,
       COUNT(*) AS count
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
GROUP BY d.city_name, o.qc_flag
ORDER BY d.city_name, o.qc_flag;
```

**Информация о чанках TimescaleDB:**
```sql
SELECT chunk_name,
       range_start,
       range_end,
       pg_size_pretty(total_bytes) AS size
FROM timescaledb_information.chunks
WHERE hypertable_name = 'staging_observations'
ORDER BY range_start DESC
LIMIT 10;
```

Выход из psql:
```sql
\q
```

## 11. Частые ошибки и их решения

### ❌ `docker: command not found`
**Причина:** Docker Desktop не установлен или не запущен.  
**Решение:** Убедись, что Docker Desktop запущен (зелёная иконка в трее) → перезапусти PowerShell.

---

### ❌ `Error response from daemon: Ports are not available`
**Причина:** Порт 8080, 3000 или 5433 уже занят другим приложением.  
**Решение:**
```powershell
# Найти, что занимает порт (например 8080)
netstat -ano | findstr :8080
# Завершить процесс по PID (замени XXXX на найденный PID)
taskkill /PID XXXX /F
```

---

### ❌ Airflow Webserver не открывается (~5 минут после старта)
**Причина:** Контейнер ещё инициализируется.  
**Решение:** Подожди 2–3 минуты. Проверь статус:
```powershell
docker compose logs -f weather_airflow_webserver
```
Жди строки: `[INFO] Listening at: http://0.0.0.0:8080`.

---

### ❌ DAG hourly_weather_etl показывает красную задачу `check_api_availability`
**Причина:** Соединение `open_meteo_api` не создано в Airflow.  
**Решение:** Создай соединение по инструкции из раздела 6.2.

---

### ❌ Задача `transform` завершается с ошибкой `could not connect to server`
**Причина:** Соединение `weather_db` не создано в Airflow.  
**Решение:** Создай соединение по инструкции из раздела 6.1.

---

### ❌ Grafana: `Data source connected, but no labels received`
**Причина:** Данных в таблицах ещё нет (конвейер не запускался или завершился с ошибкой).  
**Решение:**
1. Проверь статус DAG в Airflow — все ли задачи зелёные?
2. Выполни в psql: `SELECT COUNT(*) FROM staging_observations;` — если 0, значит ETL не отработал.
3. В Grafana установи временной диапазон `Last 7 days` и нажми **Refresh**.

---

### ❌ `WSL 2 installation is incomplete`
**Причина:** WSL2 не установлен или установлен неполностью.  
**Решение:** В PowerShell (от администратора):
```powershell
wsl --install
wsl --update
wsl --set-default-version 2
```
Перезагрузи компьютер.

---

### ❌ Очень медленный первый запуск (>30 минут)
**Причина:** Медленное интернет-соединение при скачивании образов (~5–7 ГБ).  
**Решение:** Дожди окончания. Следующие запуски будут мгновенными (образы кэшированы).

---

### ❌ `No module named 'psycopg2'` в логах Airflow
**Причина:** `requirements.txt` не установился в контейнере.  
**Решение:**
```powershell
docker compose restart weather_airflow_worker
docker compose logs weather_airflow_worker | Select-String "psycopg2"
```
