## 2. Пункт 1 — DAG ежедневного обновления архива

### Что делает
Каждый день в 02:00 UTC автоматически догружает вчерашние данные
для всех 8 городов и обновляет агрегаты TimescaleDB.
Больше не нужно запускать `historical_backfill` вручную.

### Установка

**Шаг 1.** Скопируй файл `daily_archive_update.py` в папку DAG:

```powershell
copy daily_archive_update.py C:\weather_pipeline\dags\
```

**Шаг 2.** Подожди 30 секунд — Airflow подхватит DAG автоматически.

**Шаг 3.** Открой Airflow UI → http://localhost:8080

Найди DAG **`daily_archive_update`** в списке и **активируй его** (переключатель слева → синий).

**Шаг 4.** Проверь что DAG появился и активен:

```
Статус: ● (синий переключатель)
Schedule: 0 2 * * *
Next Run: завтра в 02:00 UTC
```

### Ручной запуск для проверки

Нажми **▶ Trigger DAG** → DAG загрузит вчерашний день (192 записи = 8 городов × 24 часа).

После завершения проверь в psql:

```powershell
docker exec -it weather_pg_data psql -U weather_user -d weather_db -c "
SELECT MAX(observed_at) AS latest FROM staging_observations;
"
```

Должно показать вчерашнюю дату.

## 3. Пункт 5 — DAG проверки качества данных

### Что делает
Запускается каждый час в :15 (через 15 минут после основного ETL).
Выполняет 5 проверок и пишет результаты в таблицу `data_quality_log`.
Если критическая проверка провалилась — задача становится красной в Airflow UI.

### Установка

**Шаг 1.** Скопируй файл:

```powershell
copy data_quality_check.py C:\weather_pipeline\dags\
```

**Шаг 2.** Создай таблицу логов заранее (DAG создаст её сам при первом запуске,
но лучше сделать это явно):

```powershell
docker exec -it weather_pg_data psql -U weather_user -d weather_db -c "
CREATE TABLE IF NOT EXISTS data_quality_log (
    id            BIGSERIAL PRIMARY KEY,
    checked_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    check_id      VARCHAR(10) NOT NULL,
    check_name    VARCHAR(200) NOT NULL,
    status        VARCHAR(10) NOT NULL,
    detail        TEXT,
    rows_affected INTEGER
);
CREATE INDEX IF NOT EXISTS idx_dq_log_time
    ON data_quality_log (checked_at DESC);
"
```

**Шаг 3.** Активируй DAG в Airflow UI:

Найди **`data_quality_check`** → включи переключатель.

**Шаг 4.** Запусти вручную для проверки: **▶ Trigger DAG**

### Просмотр результатов проверок

```powershell
docker exec -it weather_pg_data psql -U weather_user -d weather_db -c "
SELECT checked_at AT TIME ZONE 'Europe/Moscow' AS time_msk,
       check_id, status, detail
FROM data_quality_log
ORDER BY checked_at DESC
LIMIT 20;
"
```

Ожидаемый вывод:
```
         time_msk          | check_id | status |              detail
---------------------------+----------+--------+----------------------------------
 2026-05-10 15:15:02+03:00 | DQ-01    | PASS   | Городов с данными: 8 из 8
 2026-05-10 15:15:02+03:00 | DQ-02    | PASS   | Записей с qc_flag=2: 0
 2026-05-10 15:15:02+03:00 | DQ-03    | PASS   | Последний бакет агрегата: ...
 2026-05-10 15:15:02+03:00 | DQ-04    | PASS   | Аномальных скачков давления: 0
 2026-05-10 15:15:02+03:00 | DQ-05    | PASS   | Записей вне диапазона: 0
```

### Добавление панели в Grafana (история проверок)

В Grafana → дашборд → **Add panel** → тип **Time series**:

```sql
SELECT
    checked_at AS time,
    CASE status WHEN 'PASS' THEN 1 ELSE 0 END AS passed,
    check_id || ': ' || check_name AS metric
FROM data_quality_log
WHERE $__timeFilter(checked_at)
ORDER BY checked_at
```

## 4. Пункт 6 — REST API (FastAPI)

### Что делает
HTTP API с документацией Swagger, доступный на http://localhost:8000/docs.
Позволяет запрашивать данные программно из любого приложения.

### Шаг 1 — Создать папку и файлы API

```powershell
mkdir C:\weather_pipeline\api
```

Скопируй `main.py` в `C:\weather_pipeline\api\main.py`.

Создай файл `C:\weather_pipeline\api\requirements.txt`:

```
fastapi==0.111.0
uvicorn==0.30.1
psycopg2-binary==2.9.9
pydantic==2.7.1
```

Создай файл `C:\weather_pipeline\api\Dockerfile`:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY main.py .

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Шаг 2 — Добавить сервис в docker-compose.yml

Открой `C:\weather_pipeline\docker-compose.yml` в VS Code.

Найди блок сервиса `grafana:` и **перед ним** (на том же уровне отступа) добавь:

```yaml
  # ── 10. FastAPI — REST API ────────────────────────────────────
  weather-api:
    build: ./api
    container_name: weather_api
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      WEATHER_DB_HOST: postgres-weather
      WEATHER_DB_PORT: 5432
      WEATHER_DB_NAME: ${POSTGRES_WEATHER_DB}
      WEATHER_DB_USER: ${POSTGRES_WEATHER_USER}
      WEATHER_DB_PASSWORD: ${POSTGRES_WEATHER_PASSWORD}
    depends_on:
      postgres-weather:
        condition: service_healthy
    networks:
      - weather_net
```

### Шаг 3 — Запустить контейнер API

```powershell
cd C:\weather_pipeline
docker compose up -d --build weather-api
```

Первый раз соберёт образ (~1 минута). Последующие запуски мгновенные.

Проверь что контейнер запустился:

```powershell
docker compose ps weather-api
```

Статус должен быть `Up`.

### Пункт 2 — Панель аномалий в Grafana

Открой http://localhost:3000 → дашборд **Weather Overview — Russia** → **Edit** (карандаш вверху справа) → **Add panel** → **Add new panel**.

**Настройка панели:**

- Тип: **Table**
- Заголовок: `🔴 Аномалии температуры (> 3σ)`

SQL-запрос (вставь в поле запроса):

```sql
WITH stats AS (
    SELECT station_id,
           AVG(avg_temperature_c)    AS mean_t,
           STDDEV(avg_temperature_c) AS std_t
    FROM mart_hourly_weather
    WHERE bucket > NOW() - INTERVAL '30 days'
    GROUP BY station_id
    HAVING STDDEV(avg_temperature_c) > 0
)
SELECT
    d.city_name                                               AS "Город",
    h.bucket                                                  AS "Время",
    ROUND(h.avg_temperature_c::NUMERIC, 1)                   AS "Температура °C",
    ROUND(s.mean_t::NUMERIC, 1)                              AS "Среднее °C",
    ROUND(
        ABS(h.avg_temperature_c - s.mean_t) / s.std_t, 1
    )                                                         AS "Отклонение σ"
FROM mart_hourly_weather h
JOIN stats s ON h.station_id = s.station_id
JOIN dim_station d ON h.station_id = d.station_id
WHERE $__timeFilter(h.bucket)
  AND ABS(h.avg_temperature_c - s.mean_t) / s.std_t > 3
ORDER BY "Отклонение σ" DESC
LIMIT 50
```

Нажми **Apply** → **Save dashboard** (Ctrl+S).

### Пункт 3 — Алерты в Grafana

**Алерт 1: «Нет свежих данных»**

Меню Grafana: **Alerting → Alert rules → New alert rule**

- Name: `Нет свежих данных по городу`
- Folder: `Weather`

В разделе **Define query and alert condition**:

Тип: **Code** → вставь SQL:

```sql
SELECT
    d.city_name,
    ROUND(
        EXTRACT(EPOCH FROM (NOW() - MAX(o.observed_at))) / 3600, 1
    ) AS hours_ago
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
GROUP BY d.city_name
HAVING MAX(o.observed_at) < NOW() - INTERVAL '2 hours'
```

- Condition: `IS ABOVE 0` (количество строк)
- Evaluate every: `5m`, For: `10m`

В разделе **Configure labels and notifications**:
- Severity: `critical`
- Contact point: выбери email или создай Telegram (см. ниже)

Нажми **Save rule and exit**.

---

**Алерт 2: «Экстремальный мороз»**

Новое правило → SQL:

```sql
SELECT COUNT(*) AS frost_count
FROM staging_observations o
WHERE o.observed_at > NOW() - INTERVAL '1 hour'
  AND o.temperature_c < -35
  AND o.qc_flag = 0
```

Condition: `IS ABOVE 0`

---

**Алерт 3: «Штормовой ветер»**

```sql
SELECT COUNT(*) AS storm_count
FROM staging_observations o
WHERE o.observed_at > NOW() - INTERVAL '1 hour'
  AND o.wind_speed_ms > 20
  AND o.qc_flag = 0
```

Condition: `IS ABOVE 0`

### Пункт 8 — Геокарта с маркерами городов

Открой дашборд → **Add panel** → **Add new panel**.

**Настройка:**

- Тип панели: **Geomap**
- Заголовок: `🗺 Текущая погода по городам России`

SQL-запрос:

```sql
SELECT
    d.latitude,
    d.longitude,
    d.city_name                        AS name,
    ROUND(o.temperature_c::NUMERIC, 1) AS temperature_c,
    ROUND(o.wind_speed_ms::NUMERIC, 1) AS wind_speed_ms,
    ROUND(o.humidity_pct::NUMERIC, 0)  AS humidity_pct,
    ROUND(o.pressure_hpa::NUMERIC, 0)  AS pressure_hpa
FROM staging_observations o
JOIN dim_station d ON o.station_id = d.station_id
WHERE o.observed_at = (
    SELECT MAX(o2.observed_at)
    FROM staging_observations o2
    WHERE o2.station_id = o.station_id
      AND o2.qc_flag = 0
)
ORDER BY d.city_name
```

**Настройка Geomap в правой панели:**

В блоке **Map layers → Layer 1**:
- Layer type: `Markers`
- Size: поле `wind_speed_ms` (размер маркера = скорость ветра)
- Color: поле `temperature_c`
- Color scheme: `Blue-Yellow-Red` (синий = холодно, красный = тепло)
- Tooltip: включи все поля

В блоке **Map view**:
- Initial view: `Fit to data` или вручную:
  - Lat: `62`, Lon: `95`, Zoom: `3`

Нажми **Apply** → **Save dashboard**.
